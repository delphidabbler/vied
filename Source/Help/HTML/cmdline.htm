Command Line Options

This topic describes the options available when Version Information Editor is started from the command line.

Window-less ("silent") operation

The program can export a resource source (.rc) file from a native version information (.vi) file without displaying a window. This option is provided to enable the program to be used in automated build systems and make files.

The required command line is: 

vied -makerc vifilename [rcfilename]

-makerc must be the first parameter and the name of the .vi file to be processed must be the second parameter. Put this name in quotes if it contains spaces. The third (optional) parameter is the name of the .rc file to be generated, again in quotes if necessary. If the third parameter is not provided the generated .rc will have the same name as the .vi file, but with the extension changed to .rc.

Once the file has been generated, or if there is an error, the program terminates. The program retrurns one of the following exit codes:

  • 0 – the program completed successfully.
  • 1 – error: the input file does not exist.
  • 2 – error: there was a problem loading the .vi file.
  • 3 – error: there was a problem converting and/or writing the .rc file.
  • 4 – error: one or more external files referenced from macros does not exist.

Default file paths

If a directory path is passed as the only parameter on the command line this is used as the default directory for all the program's file dialogue boxes.

The default directory may be overridden by the program if another path would make more sense.
