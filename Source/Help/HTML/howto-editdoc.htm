How do I ... Edit a document

You must first choose the document you wish to edit. This can either be a new document or you can open an existing document either by using the File | Open menu option or by dragging the file from Windows Explorer and dropping it on the window.

There are four different aspects of the document that can be edited.

Version information

This is the most important aspect of the document and is displayed in the main window. This is divided into three distinct parts, each of which is described below.

Version information items under each heading can be edited individually either by double clicking the item or by selecting it and choosing the Edit | Current Item menu option or pressing F2. A dialogue box suitable for editing the selected item is displayed.
Learn more.

Fixed File Information

The fixed file information items are numeric items, flags or bit-masks that define the file and product version numbers, the target operating system, the application type and some optional flags that provide further information about the build.

Translation Information

This section contains information about the language and character set used in the executable file.

String Information

This section contains string file information. There are 12 predefined entries each of which is labelled in the display. Each can take up to 128 characters of text.

The editor supports special fields and macros that can be inserted into string information items to enable the text to be automatically updated with the value of the field/macro. This is the most powerful feature of the Version Information Editor and greatly eases the updating of version information resources.
Learn how to use fields.

Limitations

Although the version information specification allows for more than one translation and a separate string file information section to be associated with each translation, this program supports only one translation and one string file information section.

Custom string file information entries are also permitted but not supported by the program.

Identifier

Although not strictly required, an identifier for the version information resource can be specified.
Learn more.

Version Information File Comments

Comments can be specified that are added to the top of saved version information files.

Resource file comments

Comments can be specified for inclusion in generated resource source files.
